- When a **new session** is created in the platform while the agent is already connected, the agent doesn't receive a `session_ready` message
- The session exists in the platform but the agent doesn't know about it
- When the platform sends a `request_state` for this new session, the agent doesn't have an ACP client for it

The fix should be:
- Handle the case where a session is received that the agent doesn't know about

  b) Or, there's already a message type that can handle new sessions being pushed

- Updates the session with the assigned port
